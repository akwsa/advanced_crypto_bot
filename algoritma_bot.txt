## 🔹 4. ALUR SIGNAL GENERATION (Detail)

======================== SIGNAL FLOW ========================

1. REQUEST SIGNAL
   (/signal <PAIR> command atau auto market scan)
   │
   ├──► GET PRICE DATA
   │     ├─ dari redis_price_cache (utama)
   │     └─ fallback: Indodax API get_ticker()
   │
   ├──► GET HISTORICAL DATA
   │     └─ dari database: db.get_price_history(pair, limit=100)
   │
   ├──► TECHNICAL ANALYSIS (TechnicalAnalysis)
   │     │
   │     ├── RSI (Relative Strength Index)
   │     │    └── Indikator overbought/oversold (0-100)
   │     │
   │     ├── MACD (Moving Average Convergence Divergence)
   │     │    └── Indikator trend (BULLISH/BEARISH/NEUTRAL)
   │     │
   │     ├── Bollinger Bands
   │     │    └── Price bands (upper, middle, lower)
   │     │
   │     ├── Moving Averages (SMA/EMA)
   │     │    └── SMA: 10, 20, 50, 100, 200
   │     │    └── EMA: 12, 26, 50
   │     │
   │     ├── ATR (Average True Range)
   │     │    └── Volatility measure
   │     │
   │     ├── Stochastic
   │     │    └── %K, %D
   │     │
   │     ├── ADX (Average Directional Index)
   │     │    └── Trend strength measurement
   │     │
   │     └── OBV (On-Balance Volume)
   │          └── Volume momentum
   │
   │   ===> OUTPUT: Dict {
   │            'strength': -1.0 s/d +1.0,
   │            'rsi': 0-100,
   │            'macd_signal': 'BULLISH/BEARISH/NEUTRAL',
   │            'recommendation': 'BUY/SELL/HOLD',
   │            'bb_position': 'UPPER/MIDDLE/LOWER',
   │            'price': float,
   │            'change_24h': float,
   │            dll
   │         }
   │
   ├──► MACHINE LEARNING PREDICTION (MLTradingModelV2)
   │     │
   │     ├── Input: 60 candle history
   │     ├── Features: 20+ technical indicators
   │     ├── Model: Random Forest + Gradient Boosting ensemble
   │     │
   │     ===> OUTPUT: Dict {
   │            'prediction': class (0/1/2),
   │            'confidence': 0.0-1.0,
   │            'signal_class': 'STRONG_BUY/BUY/HOLD/SELL/STRONG_SELL',
   │            'probabilities': [p0, p1, p2]
   │         }
   │
   ├──► SIGNAL QUALITY ENGINE (SignalQualityEngine V3)
   │     │
   │     ├── Confluence scoring
   │     │    └──TA + ML agreement check
   │     │
   │     ├── Cooldown check
   │     │    └── Recently traded? Skip if < 1 jam
   │     │
   │     └── Volume confirmation
   │          └── Volume spike detection
   │
   │     ===> OUTPUT: quality_score + is_valid_signal
   │
   └──► TRADING ENGINE (TradingEngine.generate_signal)
        │
        ├── Combine: 60% TA + 40% ML
        │     combined_strength = (ta_strength * 0.6) + (ml_strength * 0.4)
        │
        └── Apply Thresholds:
             - BUY:  strength >= 0.15
             - SELL: strength <= -0.15
             - HOLD: others

        ===> FINAL OUTPUT: {
                'signal': 'BUY'/'SELL'/'HOLD',
                'confidence': 0.0-1.0,
                'strength': -1.0 s/d +1.0,
                'reason': 'RSI oversold + MACD bullish',
                'stop_loss': price * 0.98,
                'take_profit': price * 1.05,
                'ta_signals': {...},
                'ml_prediction': {...},
                'quality_score': float
             }


## 🔹 5. AUTO-TRADING FLOW

======================== AUTO-TRADING ========================

SCHEDULER (Setiap 5 menit)
  │
  ├── Loop through WATCH_PAIRS
  │     │
  │     ├── Get latest price
  │     │    └── dari redis_price_cache / API
  │     │
  │     ├── Generate signal
  │     │    └── (ikuti signal generation flow di atas)
  │     │
  │     ├── Check confidence threshold
  │     │    └── Config.CONFIDENCE_THRESHOLD = 0.55
  │     │
  │     └── If signal BUY/SELL AND confidence >= threshold
  │         │
  │         ├── CHECK RISK RULES (RiskManager)
  │         │    │
  │         │    ├── Max position size: 25% balance
  │         │    ├── Max trades/day: 10
  │         │    ├── Daily loss limit: 5%
  │         │    ├── Risk-reward ratio: min 2:1
  │         │    │
  │         │    ===> PASS/FAIL
  │         │
  │         └── If PASS
  │              │
  │              ├── EXECUTE TRADE
  │              │    │
  │              │    ├── DRY RUN mode
  │              │    │    └── Simulasi: catat ke database
  │              │    │
  │              │    └── REAL mode
  │              │         └── API call: buy/sell
  │              │
  │              ├── SET STOP LOSS (2%)
  │              │    └── trailing_stop = price * 0.98
  │              │
  │              ├── SET TAKE PROFIT (5%)
  │              │    └── take_profit = price * 1.05
  │              │
  │              └── MONITOR POSITION
  │                   │
  │                   ├── Trailing stop activation
  │                   │    └── Activate after +2% profit
  │                   │    └── Trail by 1.5%
  │                   │
  │                   └── Auto-sell at TP
  │                        └── Jika price >= take_profit

            └── Simpan ke SQLite untuk ML training


## 🔹 7. SCHEDULED TASKS DETAIL

======================== SCHEDULER ========================

Task 1: Market Scan (5 menit interval)
  - Scan semua WATCH_PAIRS
  - Generate signal per pair
  - Cek RSI + MACD conditions
  - RSI < 40 + MACD bullish = STRONG_BUY signal
  - RSI > 65 + MACD bearish = STRONG_SELL signal
  - Queue strong signals ke signal_queue
  - Kirim notifikasi ke Telegram admins
  - Broadcast ke subscribers

Task 2: DB Cleanup (6 jam interval)
  - Hapus price data > 30 hari
  - Optimasi database
  - Log cleanup results

Task 3: Signal Stats (1 jam interval)
  - Get queue statistics
  - Log: total signals, buy signals, sell signals
  - Track signal quality over time

Task 4: Cache Health (30 menit interval)
  - Check Redis connection
  - Report local cache size
  - Log health status

## 🔹 12. ERROR HANDLING

======================== ERROR HANDLING ========================

1. API Errors
   - Timeout: Retry with exponential backoff
   - Rate limit (429): Wait and retry
   - Invalid response: Log and skip

2. Database Errors
   - Connection failed: Fallback to in-memory
   - Query error: Log and continue

3. ML Errors
   - Model not loaded: Fallback to V1
   - Training failed: Skip and log
   - Prediction error: Use default (0.5)

4. Trading Errors
   - Insufficient balance: Skip trade
   - API error: Log and retry
   - Invalid parameters: Validate and reject
