4. PROCESS FLOW DETAIL
================================================================================

┌─────────────────────────────────────────────────────────────────────────────┐
│ PROCESS: GENERATE SIGNAL (untuk /signal command)                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│ Step 1: LOAD HISTORICAL DATA                                              │
│   └─> bot.py._load_historical_data(pair)                                 │
│       ├─> Check self.historical_data[pair] (memory)                    │
│       ├─> If empty → database.get_price_history(pair, limit=200)        │
│       └─> If still empty → fetch dari API dan simpan ke DB                │
│                                                                             │
│ Step 2: TECHNICAL ANALYSIS                                              │
│   ├─> TechnicalAnalysis(df)                                           │
│   │   └─> calculate_all_indicators()                                   │
│   │       ├─> SMA (20, 50, 100, 200)                                    │
│   │       ├─> EMA (12, 26)                                             │
│   │       ├─> RSI (14)                                                 │
│   │       ├─> MACD (12, 26, 9)                                         │
│   │       ├─> Bollinger Bands (20, 2)                                 │
│   │       ├─> ATR (14)                                                 │
│   │       ├─> Stochastic (14, 3)                                      │
│   │       ├─> ADX (14)                                                │
│   │       └─> OBV, Volume SMA, Volatility                             │
│   │                                                                    │
│   └─> ta.get_signals()                                                 │
│       ├─> Analyze setiap indicator                                     │
│       ├─> Calculate strength (-1 to +1)                              │
│       └─> Return ta_signals {recommendation, strength, reason}        │
│                                                                             │
│ Step 3: ML PREDICTION                                                    │
│   ├─> ml_model.predict(df)                                            │
│   │   ├─> Prepare features                                           │
│   │   ├─> Extract indicators                                         │
│   │   ├─> Add S/R features (V2)                                     │
│   │   ├─> Add market regime (V2)                                     │
│   │   └─> Run prediction                                             │
│   │                                                                    │
│   └─> Return prediction, confidence, signal_class                      │
│                                                                             │
│ Step 4: SIGNAL QUALITY CHECK                                            │
│   ├─> signal_quality_engine.check(signal)                              │
│   │   ├─> Confluence scoring                                         │
│   │   ├─> Cooldown check                                             │
│   │   └─> Volume confirmation                                       │
│   │                                                                    │
│   └─> If quality < threshold → return HOLD                            │
│                                                                             │
│ Step 5: COMBINE TA + ML                                                 │
│   ├─> trading_engine.generate_signal()                                 │
│   │   ├─> Validate inputs                                           │
│   │   ├─> Calculate ML strength (dengan bias fix)                    │
│   │   ├─> Combine TA + ML (weighted)                                 │
│   │   ├─> Apply thresholds                                          │
│   │   └─> Return final signal                                        │
│   │                                                                    │
│   └─> Output: {recommendation, reason, ta_strength, ml_confidence,     │
│                combined_strength, price}                                │
│                                                                             │
│ Step 6: FORMAT & RETURN                                                │
│   ├─> bot._format_signal_message_html(signal)                          │
│   └─> Send ke Telegram                                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ PROCESS: EXECUTE TRADE (Auto Trading)                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│ Step 1: CHECK CONDITIONS                                              │
│   ├─> Check auto_trading_enabled                                      │
│   ├─> Check pair di auto_trade_pairs                                   │
│   └─> Check risk limits (daily loss, drawdown)                        │
│                                                                             │
│ Step 2: GET SIGNAL                                                     │
│   └─> Same seperti PROCESS: GENERATE SIGNAL                           │
│                                                                             │
│ Step 3: RISK MANAGEMENT                                                │
│   ├─> risk_manager.check_daily_loss_limit()                          │
│   ├─> risk_manager.check_drawdown()                                  │
│   ├─> risk_manager.calculate_position_risk()                         │
│   └─> If not OK → Skip trade                                           │
│                                                                             │
│ Step 4: POSITION SIZING (V3 dengan Kelly)                              │
│   ├─> ml_model_v3.get_kelly_fraction()                               │
│   ├─> Calculate position size based on Kelly                        │
│   └─> Apply Config.MAX_POSITION_SIZE limit                            │
│                                                                             │
│ Step 5: Execute ORDER                                                 │
│   ├─> indodax_api.order(pair, side, amount, price)                    │
│   ├─> If market order → instant execution                             │
│   └─> If limit order → wait untuk fill                               │
│                                                                             │
│ Step 6: POST-EXECUTION                                                 │
│   ├─> Save trade ke database                                          │
│   ├─> Update portfolio                                               │
│   ├─> Setup trailing stop ( jika enabled)                           │
│   └─> Send notification ke Telegram                                  │
│                                                                             │
│ Step 7: MONITOR POSITION                                               │
│   ├─> Watch untuk take profit / stop loss                            │
│   ├─> Activate trailing stop jika profit > X%                     │
│   ├─> Move to break-even jika profit > Y%                            │
│   └─> Execute exit jika condition met                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

│ DATA FLOW: PRICE DATA                                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Indodax API         Price Poller          Redis Cache        In-Memory      │
│  (REST API)   ───►  (15s interval)  ───►  (price_cache)  ───►  (dict)    │
│       │                                    │                   │            │
│       ▼                                    ▼                   ▼            │
│  ┌─────────────────────────────────────────────────────────────────────┐  │
│  │                    PRICE DATA FLOW                                 │  │
│  │  1. API fetch → ticker['last', 'volume', 'bid', 'ask']            │  │
│  │  2. Poller update → redis_price_cache.set_price()                 │  │
│  │  3. Cache update → bot.price_data[pair] = {...}                   │  │
│  │  4. DB save → database.save_price_data() (per 5 min)               │  │
│  │  5. Historical → bot.historical_data[pair] += new_candle          │  │
│  └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│  Usage Points:                                                          │
│  - /price command: langsung dari API                                   │
│  - /signal command: dari historical_data (min 60 candles)            │
│  - Auto-trading: dari redis_price_cache (fast)                        │
│  - Monitoring: dari price_monitor                                      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

8. ERROR HANDLING
================================================================================

┌─────────────────────────────────────────────────────────────────────────────┐
│ ERROR HANDLING STRATEGY                                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. API ERRORS (IndodaxAPI)                                              │
│     - Timeout: Retry dengan exponential backoff (2s, 4s, 6s)          │
│     - Rate Limit (429): Cooldown 60s, skip affected pair                │
│     - Invalid Pair: Add ke blacklist, skip future polls               │
│     - Network Error: Log and continue                                  │
│                                                                             │
│  2. DATABASE ERRORS                                                     │
│     - Connection Error: Retry 3x, then fallback to in-memory          │
│     - Query Error: Log error, return empty/False                       │
│     - Corruption: Recreate database if needed                           │
│                                                                             │
│  3. ML MODEL ERRORS                                                      │
│     - Load Error: Create new model                                      │
│     - Training Error: Log, continue dengan existing model             │
│     - Prediction Error: Return neutral ( HOLD)                        │
│                                                                             │
│  4. SIGNAL GENERATION ERRORS                                            │
│     - Insufficient Data: Return None, notify user                     │
│     - Invalid Data: Log, return HOLD                                   │
│     - All NaN: Return HOLD                                            │
│                                                                             │
│  5. TELEGRAM ERRORS                                                     │
│     - Send Error: Log, continue                                         │
│     - Parse Error: Send tanpa parse_mode                               │
│     - Rate Limit: Wait and retry                                      │
│                                                                             │
│  6. GRACEFUL SHUTDOWN                                                   │
│     - Catch KeyboardInterrupt                                         │
│     - Stop all background threads                                     │
│     - Save state to database                                          │
│     - Close connections                                                │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
